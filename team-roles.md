# Team Roles

## Project: Python Task Manager App

---

## Team Members & Roles

### 👤 Member A — Project Manager / Scrum Master
- Facilitates sprint ceremonies (planning, review, retrospective)
- Tracks progress and removes blockers
- Maintains the product backlog
- Communicates with stakeholders

---

### 👤 Member B — Lead Developer (Backend)
- Designs and implements core Python backend logic
- Manages database models and data layer
- Writes unit and integration tests
- Reviews pull requests from other developers

---

### 👤 Member C — Frontend / UI Developer
- Builds the user interface (CLI or web frontend)
- Integrates frontend with backend services
- Ensures usability and responsive design
- Writes frontend tests

---

### 👤 Member D — QA Engineer / DevOps
- Creates and maintains the QA plan
- Writes and runs test cases
- Manages CI/CD pipeline configuration
- Handles deployment and release processes
- Maintains the defect log

---

## Role Responsibilities Summary

| Role                  | Member   | Key Responsibility                          |
|-----------------------|----------|---------------------------------------------|
| Project Manager       | Member A | Planning, tracking, stakeholder comms       |
| Lead Backend Dev      | Member B | Python logic, database, backend tests       |
| Frontend Dev          | Member C | UI, integration, frontend tests             |
| QA Engineer / DevOps  | Member D | Testing, CI/CD, deployment, defect tracking |

---

## Team Norms

- All code must be reviewed before merging to `dev`
- Branches follow naming: `feature/us-XXX-short-description`
- Commit messages must be clear and descriptive
- Daily standups at 9:00 AM
- Blockers are raised immediately in the team chat
